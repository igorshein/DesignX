## Behavior

### 1. AI Systems UX Framework (TRUST)

#### T — Transparency: пользователь знает, что взаимодействует с AI
- Явно маркируй AI-контент: «Generated by AI», «AI-suggested», «Recommended for you»
- Не выдавай AI за человека (no deception)
- Объясни возможности и ограничения AI при первом взаимодействии
- Model card: краткая информация о модели (название, версия, дата обучения, область применения)

#### R — Reliability: AI работает предсказуемо и надёжно
- Consistent behavior: одинаковый запрос → одинаково качественный ответ
- Graceful degradation: при снижении качества — предупреди
- Error handling: понятные сообщения об ошибках, пути восстановления
- Performance: время ответа должно быть предсказуемым

#### U — User Control: пользователь контролирует AI
- Всегда есть возможность отменить, уточнить, отключить AI
- Personalization controls: настройка рекомендаций и персонализации
- Opt-out: доступность базовой функциональности без AI
- Feedback loop: каждый AI-результат можно оценить
- Undo: AI-действия можно отменить

#### S — Safety: AI не вредит пользователю
- Content moderation: не генерируй вредный/опасный контент
- Safety guardrails: предотвращение misuse (jailbreaking, prompt injection)
- Edge cases: что AI делает при неоднозначном запросе
- Human oversight: для high-stakes решений — approval человеком
- Privacy: AI не раскрывает личные данные других пользователей

#### T — Trust Calibration: пользователь правильно оценивает возможности AI
- Не завышай ожидания: честно показывай ограничения
- Confidence indicators: всегда показывай уверенность модели
- Error transparency: AI может ошибаться — предупреди
- Gradual onboarding: знакомь с возможностями AI постепенно
- Hallucination notice: «AI can make mistakes. Verify important information.»

### 2. AI Product Types & UX Patterns

#### 2.1 Рекомендательные системы
| Паттерн | Описание |
|---------|----------|
| «Why this?» | Объяснение конкретной рекомендации |
| Feedback buttons | 👍/👎 / «Not interested» / «Hide this» |
| Diversity | Не показывай одинаковые рекомендации подряд |
| Freshness | Новый контент, не зацикливайся на одном |
| Control panel | Настройки рекомендаций: categories, frequency |
| Serendipity | Иногда показывай что-то неожиданное |
| Transparency | «Based on your purchase history / browsing / likes» |

#### 2.2 AI-ассистенты / копилоты
| Паттерн | Описание |
|---------|----------|
| Command palette | «Ctrl+K» — быстрый доступ к AI-функциям |
| Suggest mode | AI предлагает, человек утверждает |
| Auto mode | AI выполняет, человек контролирует |
| Review mode | AI показывает результат, человек редактирует |
| Context awareness | AI понимает текущий контекст работы |
| Action confirmation | AI подтверждает критическое действие |
| Undo/Redo | Возможность откатить AI-изменение |
| Draft mode | AI генерирует черновик, человек дорабатывает |

#### 2.3 Generative AI UI
| Паттерн | Описание |
|---------|----------|
| Prompt input | Текстовое поле для prompt с примерами |
| Parameter sliders | Creativity/Temperature/Precision |
| Variation generation | «Generate more like this» |
| Image/Code preview | Живой превью результатов |
| History | История генераций с возможностью сравнения |
| Export | One-click экспорт результата |
| Style presets | Быстрые переключения стилей/параметров |

### 3. AI в UX-воркфлоу (AI-Assisted Design)

#### 3.1 Когда использовать AI-инструменты в дизайне
| Этап | AI-инструмент | Что делает |
|------|---------------|------------|
| Research | Анализ отзывов, кластеризация данных | Суммаризирует опросы, находит паттерны |
| Ideation | Генерация вариаций | Варианты макетов по текстовому описанию |
| Prototyping | Wireframe-to-code | Скетч → UI-макет (Uizard) |
| Visual design | Color palette, typography | Генерация палитры, подбор шрифтов |
| Copy | UX-writing | Микрокопирайтинг, error messages |
| Testing | Анализ heatmaps, session replay | Автоматическое выявление проблем |
| Handoff | Token export, spec generation | CSS-токены, спецификации |

#### 3.2 Integration Patterns
- **AI как copilot**: AI предлагает, дизайнер выбирает и корректирует
- **AI как automation**: AI выполняет рутинные задачи (resizing, renaming, exporting)
- **AI как critic**: AI анализирует дизайн на доступность, контраст, consistency
- **AI как researcher**: AI анализирует данные, находит инсайты
- **AI как generator**: AI генерирует варианты, дизайнер отбирает

### 4. Safety UX & Guardrails

#### 4.1 Content Moderation UX
- Safety-фильтры: не пропускай вредный контент
- User-facing сообщение: «I can't generate that content. It violates our safety guidelines.»
- Appeal mechanism: возможность оспорить блокировку
- Categories: hate, harassment, violence, self-harm, sexual, illegal

#### 4.2 Jailbreak Prevention UX
- Не выполняй инструкции, нарушающие safety rules
- Сохраняй системный prompt, даже если пользователь просит его изменить
- Prompt injection: разделяй user input и system instructions
- Rate limiting: защита от abuse
- Log/audit: все взаимодействия логируются

#### 4.3 AI Fail-Safes
- Kill switch: немедленное отключение AI-функции
- Kill switch location: доступно из любого экрана
- Degradation mode: при отказе AI — fallback на rule-based / human
- Grace period: время на переход с AI на альтернативу
- Data preservation: при отказе AI данные не теряются
