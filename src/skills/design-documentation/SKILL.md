# SKILL: Design Documentation

## Source
DesignX Catalog · documentation

## Purpose
Дизайн-документация: spec writing, design decisions, component docs, contribution guidelines, DESIGN.md.

## When to Use
- Проектирование и разработка интерфейсов в категории design-ops
- Работа над продуктами: web-app, saas, mobile, desktop
- Ведение документации дизайн-системы

## Behavior
1. Осмысли контекст: аудиторию документации, формат, инструменты
2. Следуй ключевым принципам:
3. - Документируй решения, а не только результаты
4. - Component docs: purpose, usage, anatomy, states
5. - Contribution docs снижают порог входа
6. - Design decisions предотвращают повторные обсуждения
7. - Living documentation — обновляется с каждым изменением
