# SKILL: Design QA Process

## Source
`DesignX`

## Purpose
Приёмка дизайна: pixel-perfect проверка, QA checklist, design acceptance criteria, sign-off workflow.

## When to Use
- Проектирование и разработка интерфейсов в категории critique
- Работа над продуктами: web-app, saas, mobile, desktop, ecommerce
- Контроль качества перед релизом

## Behavior
1. Осмысли контекст: макеты, реализацию, требования, окружение
2. Следуй ключевым принципам:
3. - QA каждого состояния, а только happy path
4. - Pixel-perfect = допуск ±1px
5. - Проверяй каждый responsive breakpoint
6. - Accessibility — требование QA
7. - Фиксируй баги со скриншотами и спецификацией
