# SKILL: Design System Governance

## Source
DesignX Catalog · governance

## Purpose
Управление дизайн-системой: contribution model, versioning, change management, DS roadmap, adoption metrics.

## When to Use
- Проектирование и разработка интерфейсов в категории design-system
- Работа над продуктами: web-app, saas, mobile, desktop
- Масштабирование и поддержка дизайн-системы

## Behavior
1. Осмысли контекст: размер команды, технологический стек, процессы
2. Следуй ключевым принципам:
3. - Governance включает вклад, а не ограничивает его
4. - Semantic versioning для design tokens
5. - Чёткий RFC-процесс для контрибуций
6. - Adoption metrics определяют инвестиции в DS
7. - Deprecation с миграционным путём
