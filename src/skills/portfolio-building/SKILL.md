## Behavior

### 1. Case Study Structure (Problem → Process → Solution → Impact)

```markdown
## Case Study: [Project Name]

### Context
- **Product**: [name, type, platform]
- **My Role**: [UX researcher / UI designer / Product designer / Design lead]
- **Timeline**: [3 months / 2 sprints / etc.]
- **Team**: [size, composition]
- **Tools**: [Figma, Miro, Amplitude, etc.]

### Problem
- **Business problem**: [what the business needed]
- **User problem**: [what users struggled with]
- **Hypothesis**: [what we believed would help]
- **Constraints**: [time, budget, technical, regulatory]

### Process

#### Research
- Methods used: [interviews, surveys, analytics, competitive analysis]
- Key findings: [3-5 bullet points]
- How research shaped direction: [specific example]

#### Ideation
- Concepts explored: [2-3 approaches]
- Why chosen direction: [rationale with evidence]
- What was rejected: [and why]

#### Prototyping
- Fidelity: [low → hi, why]
- Key iterations: [what changed between versions]
- User testing: [methods, findings, changes made]

#### Visual Design
- Design decisions: [color, typography, layout choices]
- Design system: [how DS was used or extended]
- States covered: [default, hover, active, disabled, loading, empty, error]

### Solution
- **Final design**: [screenshots / prototype link]
- **Key features**: [top 3-5]
- **Accessibility**: [WCAG level, specific accommodations]
- **Edge cases handled**: [specific examples]

### Impact
- **Metrics**: [before → after: conversion, NPS, task success, time-on-task]
- **Business impact**: [revenue, retention, cost savings]
- **User impact**: [satisfaction, feedback quotes]
- **Team impact**: [DS adoption, process improvements]

### Learnings
- What went well
- What I'd do differently
- Key takeaways

### Links
- [Live product]
- [Prototype]
- [Design system]
```

### 2. Case Study Templates by Role

#### 2.1 Product Designer
- Фокус на full product process: research → ideation → prototyping → delivery
- Key: связь дизайн-решений с бизнес-метриками
- Примеры: end-to-end фичи, redesign продуктов, new product launch

#### 2.2 UX Researcher
- Фокус на исследовательские методы и выводы
- Key: методология, sample size, findings → recommendations
- Примеры: usability testing, field studies, surveys, analytics deep-dives

#### 2.3 UI / Visual Designer
- Фокус на визуальные решения и дизайн-системы
- Key: design system contribution, visual consistency, states, accessibility
- Примеры: design system component, marketing site, visual refresh

#### 2.4 UX Writer / Content Designer
- Фокус на tone of voice, microcopy, content strategy
- Key: before/after copy, A/B test results, content model
- Примеры: error message system, onboarding copy, voice & tone guide

#### 2.5 Design Lead / Manager
- Фокус на team leadership, process, impact
- Key: team growth, process improvement, strategic impact
- Примеры: design system adoption, team scaling, design culture

### 3. Portfolio Checklist

#### 3.1 Content
- [ ] 3-5 case studies (разные типы проектов, платформы)
- [ ] Каждый кейс: context, problem, process, solution, impact
- [ ] Роль чётко указана
- [ ] Metrics: before → after с цифрами
- [ ] Process: research → ideation → testing → delivery (не только финал)
- [ ] States: не только happy path, но и empty/error/edge cases
- [ ] Accessibility: WCAG уровень, конкретные решения

#### 3.2 Presentation
- [ ] Clean, minimal layout (fokus на контент, не на шаблон)
- [ ] High-quality screenshots (не размытые, 2x)
- [ ] Прототип или live demo доступны
- [ ] Loading < 3s, mobile-friendly
- [ ] Контактная информация: email, LinkedIn, GitHub

#### 3.3 Storytelling
- [ ] Clear narrative: problem → solution → impact
- [ ] Design decisions обоснованы (почему, а не что)
- [ ] User research повлияла на дизайн (показать как)
- [ ] What I learned: рефлексия, growth

### 4. Interview Preparation

#### 4.1 Portfolio Presentation
- Структура: 20-30 минут презентации + 10-15 минут Q&A
- 3-5 минут на контекст, 10-15 на процесс, 5-10 на результаты
- Practice: проговори вслух, тайминг
- Anticipate questions: почему выбрал этот подход, что было бы иначе

#### 4.2 Whiteboard Challenge
- Фреймворк: Clarify → Ideate → Decide → Prototype → Explain
- Think aloud: говори, о чём думаешь
- User-first: начинай с пользователя, не с UI
- Trade-offs: покажи, что понимаешь ограничения

#### 4.3 Common Interview Questions
- «Walk me through your portfolio» — structure заранее
- «Tell me about a time you disagreed with a stakeholder» — STAR method
- «How do you handle ambiguity?» — example with process
- «What's your design process?» — не generic, а твой реальный процесс
- «How do you measure design success?» — metrics-driven thinking
