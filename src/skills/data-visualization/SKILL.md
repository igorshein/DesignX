# SKILL: Data Visualization Design

## Source
`designer-skills` · [GitHub](https://github.com/Owl-Listener/designer-skills)

## Purpose
Проектирование data-heavy интерфейсов: дашборды, KPI-панели, графики, таблицы. Выбор правильного типа визуализации под задачу, иерархия данных, адаптивность, доступность.

## When to Use
- Дашборды, аналитические панели, reporting modules
- Любой интерфейс с числами, трендами, сравнениями
- Продукты: dashboard, web-app, saas, analytics platform

## Behavior
1. Определи задачу пользователя: сравнение, тренд, распределение, состав, связь?
2. Выбери тип визуализации под задачу:
   - Тренд во времени → line chart (НЕ bar chart для >12 точек)
   - Сравнение категорий → bar chart (horizontal для длинных подписей)
   - Состав → stacked bar / treemap (НЕ pie chart если >5 категорий)
   - Распределение → histogram / box plot
   - Связь → scatter plot + trend line
   - KPI → big number + sparkline, не прячь в график
3. Иерархия: самое важное — наверх (KPI-ряд), детали — под графиками.
4. Цифры: tabular figures (одинаковая ширина цифр) для чисел в таблицах.
5. Цвет: не полагаться только на цвет — добавляй паттерны/текст. Акцент ровно на одном ряде данных.
6. Состояния: loading (skeleton для графиков), empty (нет данных за период), error (ошибка API + кнопка повтора).
7. Адаптивность: на mobile графики в полную ширину, таблицы — горизонтальный scroll.
8. Доступность: alt-текст с выводами для каждого графика, aria-label для интерактивных элементов.

## Tags

## For Product Types
dashboard, web-app, saas, desktop

## Forbidden
- Pie chart с >5 категориями
- 3D-графики (искажают восприятие)
- Использовать цвет как единственный способ различить ряды
- Графики без контекста (ось без подписей, нет единиц измерения)
- Перегружать дашборд: >1 графика на экран без иерархии
