# SKILL: Component Naming Standards

## Source
`DesignX`

## Purpose
Стандарты именования компонентов: DS taxonomy, naming conventions, layer organization, auto-layout naming.

## When to Use
- Проектирование и разработка интерфейсов в категории design-system
- Работа над продуктами: web-app, saas, mobile, desktop
- Организация библиотеки компонентов

## Behavior
1. Осмысли контекст: размер DS, команду, инструменты
2. Следуй ключевым принципам:
3. - Консистентное именование — тоже design token
4. - Именуй по назначению, а не по внешнему виду
5. - Variant naming: Base > Type > Size > State
6. - Имена слоёв совпадают с анатомией компонента
7. - Auto-layout naming для передачи разработчикам
