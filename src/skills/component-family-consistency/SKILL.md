# Component Family Consistency

## Source
DesignX Catalog · design-system

## Purpose
Shared radius, colour, height for related components (buttons, inputs, pills). Visual alignment, consistent padding, font sizes within families.

## When to Use
- Проектирование и разработка интерфейсов в категории design-system
- Работа над продуктами: web-app, saas, mobile, desktop
- Интеграция с дизайн-системами и ИИ-агентами

## Behavior
1. Осмысли контекст: тип продукта, аудиторию, платформу, технические ограничения
2. Следуй ключевым принципам:
   Related components share visual properties
   Same height for buttons and inputs in same row
   Consistent border-radius across family
   Shared color semantics across family
   Match padding and font sizes within family
