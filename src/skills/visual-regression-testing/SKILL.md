# SKILL: Visual Regression Testing

## Source
DesignX Catalog · testing

## Purpose
Визуальное регрессионное тестирование: Percy, Chromatic, screenshot diff, baseline management, CI-интеграция.

## When to Use
- Проектирование и разработка интерфейсов в категории critique
- Работа над продуктами: web-app, saas, mobile, desktop
- Контроль качества визуальных изменений

## Behavior
1. Осмысли контекст: компоненты, страницы, браузеры, типы изменений
2. Следуй ключевым принципам:
3. - Baseline для каждого компонента, diff для каждого изменения
4. - Настрой порог для игнорирования anti-aliasing
5. - Компонентные тесты ловят проблемы на раннем этапе
6. - Кросс-браузерные скриншоты ловят рендеринг-баги
7. - Ревью всех diff перед approve
