# SKILL: Design Systems Architecture

## Source
DesignX Catalog · design-system

## Purpose
Архитектура дизайн-систем для масштабирования: shared vs local tokens, versioning, npm distribution, federation portal, contribution model.

## When to Use
- Проектирование и разработка интерфейсов в категории design-system
- Работа над продуктами: web-app, saas, mobile, ecommerce, desktop
- Интеграция с дизайн-системами и ИИ-агентами

## Behavior
1. Осмысли контекст: тип продукта, аудиторию, платформу, технические ограничения
2. Следуй ключевым принципам:
3. - Multi-brand: базовые токены + override для бренда
4. - Distribution: npm пакеты с semver
5. - Federation: Core (shared) + Domain (specific)
6. - Versioning: tokens major, components minor
