# SKILL: Dark Patterns Audit & Ethical Alternatives

## Source
`designer-skills, RESOURCES.md §2`

## Purpose
Полная таксономия тёмных паттернов (12 типов по Бригнуллу), их распознавание, анализ и замена этичными альтернативами. Включает чеклисты этического дизайна, integrity-фреймворк, принципы доверия и прозрачности для AI-систем.

## When to Use
- Аудит существующих интерфейсов на манипулятивные паттерны
- Проектирование новых интерфейсов с этическими ограничениями
- Ревью дизайн-систем и компонентных библиотек
- Проектирование AI-интерфейсов (доверие, explainability, контроль)
- Продукты: ecommerce, web-app, mobile, saas, landing, dashboard

## Behavior

### 1. Полная таксономия тёмных паттернов (12 типов Бригнулла)

#### 1.1 Trick Questions (Вопросы-ловушки)
- **Суть**: Формулировка вопроса намеренно запутывает, подталкивая к невыгодному выбору
- **Пример**: Двойное отрицание в чекбоксе отписки; согласие на рассылку = подписка на платный тариф
- **Альтернатива**: Прямые формулировки, affirmative consent, разделение действий на разные шаги

#### 1.2 Sneaking (Скрытые опции)
- **Суть**: Платные услуги, галочки или согласия незаметно добавляются в поток
- **Пример**: Автоматическое добавление страховки в корзину; pre-ticked checkbox на email-маркетинг
- **Альтернатива**: Все опции явные, pre-ticked только критически необходимые, сумма до подтверждения

#### 1.3 Urgency (Ложная срочность)
- **Суть**: Искусственное ограничение времени или количества для давления
- **Пример**: «Только 3 места по этой цене», «Предложение истекает через 00:05:00»
- **Альтернатива**: Реальная срочность (если есть); честное указание остатка; отсутствие фейковых таймеров

#### 1.4 Misdirection (Направленное отвлечение)
- **Суть**: Визуальное акцентирование выгодного компании выбора при скрытии альтернатив
- **Пример**: Яркая кнопка «Принять все cookie», серая ссылка «Настроить»; cancel-кнопка невидима
- **Альтернатива**: Равный визуальный вес для всех опций; одинаковый размер и контраст

#### 1.5 Social Proof (Манипуляция социальным доказательством)
- **Суть**: Фальшивые или вводящие в заблуждение отзывы, уведомления о действиях других
- **Пример**: «Иван только что купил этот товар» (бот); «23 человека смотрят эту страницу»
- **Альтернатива**: Реальные отзывы с верификацией; прозрачная модерация; честная статистика

#### 1.6 Forced Action (Принудительное действие)
- **Суть**: Для отказа от одного действия требуется совершить другое, нежелательное
- **Пример**: Обязательный ввод номера телефона для отказа от рассылки; подписка для чтения статьи
- **Альтернатива**: Отписка в один клик без дополнительных данных; free tier без принуждения

#### 1.7 Confirmshaming (Стыд за отказ)
- **Суть**: Формулировка отказы вызывает чувство вины or стыда
- **Пример**: «Нет, я не хочу экономить деньги»; «Я предпочитаю оставаться невежественным»
- **Альтернатива**: Нейтральные формулировки: «Нет, спасибо»; уважение к любому выбору

#### 1.8 Nagging (Навязчивые уведомления)
- **Суть**: Повторяющиеся запросы, которые невозможно отключить окончательно
- **Пример**: Pop-up «Оцените приложение» после каждого действия; повторный запрос push
- **Альтернатива**: Один запрос в контексте; настройка частоты; «Больше не спрашивать»

#### 1.9 Friend Spam (Спам от имени пользователя)
- **Суть**: Сервис рассылает приглашения от имени пользователя без явного согласия
- **Пример**: Импорт контактов с автоматической рассылкой; скрытая галочка «Пригласить друзей»
- **Альтернатива**: Явный выбор контактов; подтверждение перед отправкой; preview сообщения

#### 1.10 Hidden Costs (Скрытые платежи)
- **Суть**: Реальная стоимость раскрывается только на финальном шаге
- **Пример**: Доставка, налоги, сервисный сбор добавляются в корзину перед оплатой
- **Альтернатива**: Полная стоимость с первого экрана; breakdown всех сборов; price transparency

#### 1.11 Bait and Switch (Приманка и подмена)
- **Суть**: Обещанное действие приводит к неожиданному результату
- **Пример**: Кнопка «Отключить звук» запускает рекламу; ссылка «Закрыть» ведёт на подписку
- **Альтернатива**: Предсказуемый результат каждого действия; явный label без обмана

#### 1.12 Roach Motel (Лёгкий вход — сложный выход)
- **Суть**: Зарегистрироваться просто, отменить подписку или удалить аккаунт — почти невозможно
- **Пример**: 3-шаговая регистрация vs звонок в поддержку для отмены; скрытая кнопка удаления
- **Альтернатива**: Self-service отмена; «отписаться в один клик»; удаление аккаунта как first-class feature

### 2. Integrity-фреймворк для проверки решений

Перед выдачей каждого интерфейсного решения пройди чеклист:

- [ ] Все ли опции имеют равный визуальный вес?
- [ ] Формулировки нейтральны и не стыдят за отказ?
- [ ] Стоимость полностью раскрыта до подтверждения?
- [ ] Отмена/отписка симметричны по сложности с подпиской?
- [ ] Пользователь явно соглашается на каждое платное действие?
- [ ] Уведомления имеют кнопку «Больше не показывать»?
- [ ] Авто-продление требует явного подтверждения?
- [ ] Реальные таймеры/лимиты — не фейковые?
- [ ] Социальное доказательство верифицировано?
- [ ] Нет pre-ticked checkbox без явной необходимости?

### 3. Принципы ответственного дизайна (Privacy & Transparency)

#### 3.1 Consent Design
- Явное информированное согласие (не pre-ticked, не подразумеваемое)
- Layered notices: кратко -> детально -> полная политика
- Возможность отзыва согласия с той же лёгкостью, что и предоставление
- Минимизация собираемых данных (data minimization)
- Гранулярный контроль: согласие на каждую категорию отдельно

#### 3.2 Privacy by Design
- Privacy — не compliance checklist, а архитектурный принцип
- Данные — first-class feature: удаление, экспорт, дашборд
- Принцип «what you see is what is collected»: UI показывает, какие данные собираются
- Асимметрия: согласие = 1 клик, отказ = 1 клик (одинаково)

#### 3.3 Trustworthy AI UX (GLASS+)
- Confidence indicators: показывай уровень уверенности AI
- Source attribution: откуда взялся ответ/рекомендация
- Explainability: почему показан именно этот контент
- Control: возможность скрыть/уточнить/отменить AI-действие
- Calibration: не обещай точности там, где её нет (honest uncertainty)
