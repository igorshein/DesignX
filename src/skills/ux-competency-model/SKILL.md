# SKILL: UX Competency Model & Career Paths

## Source

## Purpose
Компетентностные модели для UX-специалистов: грейды (junior → middle → senior → lead → principal), skills matrix по ролям, карьерные треки (IC vs management), индивидуальные планы развития.

## Behavior

### 1. Роли и грейды в UX

#### 1.1 Роли
| Роль | Фокус | Основные навыки |
|------|-------|-----------------|
| UX Researcher | Исследования | User interviews, usability testing, surveys, analytics, synthesis |
| UI / Visual Designer | Визуальный дизайн | Layout, typography, color, iconography, design systems |
| Product Designer | Full-stack | Research → UI → prototyping → delivery, product thinking |
| UX Writer | Контент | Microcopy, voice & tone, content strategy, localization |
| Design Lead | Лидерство | Team management, strategy, stakeholder management, mentorship |
| DesignOps | Процессы | Process design, tooling, governance, metrics, scaling |

#### 1.2 Грейды (Product Designer track)

| Измерение | Junior (0-2 года) | Middle (2-5 лет) | Senior (5-8 лет) | Lead (8+ лет) |
|-----------|-------------------|------------------|------------------|---------------|
| **Research** | Проводит под руководством | Самостоятельно, выбирает методы | Менторит, внедряет процессы | Определяет research strategy |
| **Visual** | Базовые макеты по DS | Создаёт компоненты DS | DS architecture, tokens | Визуальная стратегия продукта |
| **Prototyping** | Low-fi, базовые flows | Hi-fi, интерактивные прототипы | Сложные flows, микровзаимодействия | Design tooling strategy |
| **User Focus** | Исполняет задачи | Защищает пользователя | User advocacy + business | User vision для продукта |
| **Collaboration** | Работает в команде | Ведёт фичи с dev/PM | Кросс-функциональный лидер | Организует дизайн-культуру |
| **Strategy** | — | Участвует в discovery | Product strategy contribution | Определяет design direction |
| **Business** | Понимает метрики | Data-informed решения | Metrics-driven, ROI дизайна | Дизайн как бизнес-драйвер |
| **Mentorship** | — | Помогает junior | Ведёт 1-2 junior/middle | Растит команду, культуру |

### 3. Карьерные треки

#### 3.1 Individual Contributor (IC)
```
Junior → Middle → Senior → Staff → Principal
```
- Фокус на глубина экспертизы, сложность задач
- Staff/Principal: стратегическое влияние без управления людьми
- Пример: Principal Product Designer в FAANG

#### 3.2 Management Track
```
Senior → Lead → Manager → Director → VP
```
- Фокус на рост команды, процессы, культура
- Manager: 3-8 дизайнеров, Director: 8-30, VP: 30+
- Пример: Design Director в продуктовой компании

#### 3.3 Hybrid (IC + Management)
```
Senior → Lead (Player-Coach)
```
- 50-70% делает сам + 30-50% manages
- Обычно Lead / Head of Design в小さких командах

### 4. Индивидуальный план развития (IDP)

#### 4.1 Структура IDP
```yaml
idp:
  current_level: "middle"
  target_level: "senior"
  timeline: "12 months"
  
  strengths:
    - "Visual design"
    - "Prototyping"
    - "User testing"
  
  growth_areas:
    - "Product strategy"
    - "Data analysis"
    - "Stakeholder management"
  
  action_plan:
    - area: "Product strategy"
      actions:
        - "Прочитать Inspired by Marty Cagan"
        - "Провести 3 discovery-спринта"
        - "Презентовать OKR на квартал"
      timeline: "Q1-Q2"
      evidence: "OKR doc, sprint outcome"
    
    - area: "Data analysis"
      actions:
        - "Курс по Amplitude/Mixpanel"
        - "Сделать 2 UX audit с данными"
      timeline: "Q2-Q3"
      evidence: "Audit reports"
  
  mentoring:
    mentor: "Senior/Lead"
    frequency: "bi-weekly 1:1"
    focus: "Strategy thinking, stakeholder communication"
```

#### 4.2 Как агент строит IDP
1. Оцени текущий уровень (self-assessment + portfolio review)
2. Определи целевой уровень и трек (IC vs management)
3. Выяви пробелы в skills matrix
4. Подбери образовательные ресурсы из каталога
5. Составь action plan с конкретными шагами
6. Назначь checkpoints (3/6/9/12 месяцев)
