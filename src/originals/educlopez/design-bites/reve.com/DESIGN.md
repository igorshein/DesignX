# Reve — Design Language

## 1. Visual Theme & Atmosphere
Reve is a thoughtfully engineered product with a design system that reflects genuine craft. The site uses two custom proprietary fonts (ReveDisplay and ReveUI) and an 8px rhythm base — every spacing decision derived from that grid unit. The colour system is built with `light-dark()` CSS function for first-class dark mode support across a comprehensive neutral scale. The overall aesthetic is precise, scalable, and system-driven — the mark of a team that thinks in design tokens.

## 2. Color Palette & Roles
- **Neutrals** — comprehensive `light-dark()` scale from `--sl-color-gray-50` to `--sl-color-gray-950`: both light and dark values built-in
- **Light mode surface** — `#f9f9f9` to white for layered backgrounds

- **Text light mode** — near-black (`#292929`); dark mode — near-white (`#d6d6d6`)
- **Alert/status colours** — red, green, yellow, blue: full semantic colour set via `--sl-color-*`
- **Rhythm unit** — `--rv-rhythm: 8px`: all spacing is multiples of this base

The system is neutral-first, semantic-complete, and dual-mode by design.

## 7. Do's and Don'ts
**Do:** Always use multiples of 8px (`--rv-rhythm`) for all spacing decisions.
**Do:** Use `light-dark()` for every colour to ensure automatic dark mode support.
**Do:** Keep ReveDisplay for headings, ReveUI for interface text — the families are not interchangeable.
**Don't:** Introduce external typefaces — the custom family is the brand.
**Don't:** Use arbitrary spacing values — only rhythm multiples (8, 16, 24, 32, 40, 48…).
**Don't:** Define single-mode colours — every value must have both light and dark equivalents.

## 8. Responsive Behavior
- 8px rhythm grid scales naturally across viewports
- `light-dark()` handles colour scheme automatically via `prefers-color-scheme`
- Semantic colour slots (`--sl-color-*`) adapt consistently through the scale
- Font sizes likely reduce moderately at mobile; layout collapses to single column
