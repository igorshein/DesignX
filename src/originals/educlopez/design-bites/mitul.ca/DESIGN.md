# mitul.ca — Design System

## 1. Visual Theme & Atmosphere

Mitul.ca is a personal portfolio with a **print-like precision** — a white canvas, a single deep navy blue (#021093) that carries both navigation and footer weight, and Chico as the sole typeface across all roles. The compression is intentional: a font built for high-contrast editorial settings, set at the same size for nearly every element. The effect reads as confident and typographically disciplined, more zine than startup landing page. Absence of shadows, gradients, or decorative treatments keeps attention on the work itself.

## 2. Color Palette & Roles

| Role | Value | Usage |
|------|-------|-------|
| Page Background | transparent / white | Canvas |
| Primary Text | `#000000` | Body, headings |
| Navy Accent | `#021093` | Navigation, links, footer, CTAs |
| Footer Text | `rgb(252, 252, 252)` | Text on navy surfaces |
| Mid-Gray Surface | `rgb(217, 217, 217)` | Subtle card/panel backgrounds |

Two working colors: black and navy. Gray appears only as a background tint. This is a portfolio that trusts the work over the palette.

## 3. Typography Rules

- **Font family**: Chico (custom, single typeface) — high-contrast editorial serif-adjacent display face
- **Weight range**: 400 (body) and 500 (headings, buttons, links) — only two weights, minimal hierarchy
- **H1**: 24px / 36px — surprisingly compact for a portfolio hero; intimacy over grandeur
- **Body / H2–H3**: 14px / 21px — consistent 1.5x line height; legible and economical
- **No letter-spacing overrides, no transforms** — Chico sets its own tone

| Level | Size | Weight | Line Height |
|-------|------|--------|-------------|
| H1 | 24px | 500 | 36px |
| H2 / H3 / Body / Links | 14px | 400–500 | 21px |
| Buttons | 14px | 500 | 21px |

## 4. Component Stylings

- **Buttons**: Navy fill (#021093), near-white text — tight 2–4px radius, no softness
- **Navigation**: Transparent background, navy text — minimal chrome
- **Footer**: Full navy block, off-white text — clear inversion
- **Cards / panels**: Light gray surfaces (rgb(217,217,217)) — elevation by tint, not shadow
- **Links**: Navy-colored, consistent with navigation language

## 5. Layout Principles

- **Stack**: Next.js + Tailwind CSS — utility-first, component-based
- **Breakpoints**: Tailwind defaults (likely 640 / 768 / 1024 / 1280)
- **Spacing**: Tailwind 4px base unit scale
- **Border radius**: 2px and 4px only — sharp, precise edges throughout
- **Content density**: Portfolio-paced — breathing room between case study entries

## 6. Depth & Elevation

- Extremely flat — no box shadows detected
- Elevation expressed exclusively via background color shift
- Gray panels vs. white canvas create sufficient hierarchy without depth tokens

## 7. Do's and Don'ts

**Do:**
- Use Chico as the single typeface across all roles
- Maintain the two-color system: black and navy only
- Keep border radii at 2–4px — no rounded-lg or pill shapes
- Use the navy inversion pattern (white text on #021093) for emphasis sections

**Don't:**
- Introduce a third type weight or a decorative display font alongside Chico
- Add drop shadows — the system reads flat by design
- Use large rounded corners — they conflict with the print-precise character

## 8. Responsive Behavior

- Tailwind responsive breakpoints (640 / 768 / 1024 / 1280)
- Portfolio grid likely collapses to single column on mobile
- Navigation likely collapses to hamburger or inline stack
- 2–4px radii remain consistent across all breakpoints
