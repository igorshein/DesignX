# fal.ai — Design Reference

## 1. Visual Theme & Atmosphere

fal.ai presents as a next-generation AI infrastructure platform with a design language balancing technical precision and organic softness. The hero deploys **Focal** — a custom display face — at 120px with aggressive negative tracking (−4.8px), creating a compressed, forward-leaning energy. A sky-blue footer (`rgb(153,236,255)`) grounds the page in a signature color that feels both technical and approachable.

The supporting palette draws from nature-adjacent pastels — chartreuse, lavender, misty blue — distributed across content sections, giving each zone a distinct but harmonious identity. Built on Next.js + Tailwind with a meticulously defined design token system.

## 3. Typography Rules

Two custom fonts: **Focal** for display, **Public Sans Rounded** for UI and body.

| Element | Family | Size | Weight | Notes |
|---------|--------|------|--------|-------|
| H1 | Focal | 120px | 500 | −4.8px tracking, 100px line height (compressed) |
| H4 / Feature | Focal | 80px | 500 | Large secondary display |
| H3 | Focal | 36px | 500 | −1.44px tracking |
| H2 | Focal | 24px | 500 | −0.48px tracking |
| Body | Public Sans Rounded | 16px | 400 | 24px line height |
| Para | Focal | 18px | 400 | 28px line height |

Focal dominates all display levels. Public Sans Rounded handles only UI-level body text — a clean separation between editorial display and functional prose.

## 4. Component Stylings

- **Navbar:** White surface, separates from transparent header on scroll
- **Buttons:** Small 4–6px border-radius — slightly rounded, not pill-shaped
- **Cards:** Pastel background zones, no shadows
- **Code blocks:** Dark gray text on light surface
- **Feature sections:** Each gets a unique pastel background
- **Footer:** Full-bleed signature blue (`rgb(153,236,255)`)
- **Sticky nav:** Semi-transparent with backdrop blur using the signature blue tint

## 6. Depth & Elevation

No box shadows detected. Depth is achieved through:
- Background color zoning (each section = unique pastel)
- Sticky nav with backdrop-filter blur + 50% opacity signature blue
- Token-based surface stepping (surface-50 through surface-300)

## 7. Do's and Don'ts

**Do:**
- Use Focal at extreme scale with tight negative tracking for heroes
- Zone sections with distinct pastel backgrounds from the palette
- Use the signature blue (`rgb(153,236,255)`) for the footer and nav accents
- Keep border-radius small (4–6px) — not pill-shaped
- Use Public Sans Rounded only for body/UI, Focal for everything display

**Don't:**
- Use shadows — this is a flat, color-zoned system
- Mix the CTA purple with pastel backgrounds in the same section
- Create pill-shaped buttons — small radius only
- Use warm tones — the palette is cool-to-neutral throughout

## 8. Responsive Behavior

- 7 breakpoints enable fine-grained responsive control
- Hero headline scales from 120px down through each breakpoint
- Pastel section backgrounds are preserved at all widths
- Mobile layout collapses to single column with reduced padding
