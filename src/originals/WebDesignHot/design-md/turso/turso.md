## 1. Visual Theme & Atmosphere

Turso's home page opens on a `#0f1419` canvas ‚Äî a deep teal-black that
sits between Neon's pure near-black and Supabase's forest-green. The
brand cyan `#00face` (the hex literally reads "FACE") carries every
primary action and live indicator. The page reads like the marketing
equivalent of a glowing-edge-node dashboard. The first impression is
**SQLite at the edge, rendered as CRT phosphor** ‚Äî a brand that took
the metaphor seriously enough to make it the substrate.

The supporting emerald palette (`#1ebca1`, `#23765f`, `#1e6952`,
`#29886e`) gives Turso four shades of green-teal for tinted panels,
badge backgrounds, and feature blocks ‚Äî a chromatic ladder broader
than most single-accent dev brands. This breadth is what differentiates
Turso from Neon: Neon's mint is a single-accent commitment, while
Turso's cyan-emerald scale offers a tonal vocabulary across the whole
chromatic family.

The mood is **glowing edge node**. The cyan accent reads as a literal
status indicator ‚Äî when it appears next to a region label, you read
it as "live replica online." Mono numerals carry replication metrics
(latency to nearest replica, region count, query latency); the
emerald-tinted feature panels function as section emphasis without
breaking from the canvas's chromatic family.

There is no skeuomorphic 3D, no decorative gradient mesh, no neon-
glow effect outside the focus ring. Turso's brand identity comes from
chromatic discipline (cyan + emerald + near-black) and typographic
restraint (Inter + mono, no display experiments). The brand-as-pun
(`#00face`) is the wink that tells you a developer designed this
brand for developers.

**Key Characteristics**

- Deep teal-black canvas (`#0f1419`) with CRT-bright cyan accent (`#00face`)
- Brand hex literally spells "FACE" ‚Äî a self-referential developer wink
- Emerald tonal scale (`#1ebca1`, `#23765f`, `#1e6952`, `#29886e`) for tinted panels
- Inter display + body, Fira Code or JetBrains Mono for code/metrics
- 8px button radius + 12px card radius (Vercel-class geometry)
- Cyan reserved for primary action and live indicators only
- Mono with `tnum` and `zero` features for replication metrics
- Edge-node map visualization as signature layout element
- Dark-only ‚Äî no light variant
- Region pills using mono labels with cyan dot indicators

### Principles

- **Mono with `tnum` for metrics**: replication latency, region count, query time ‚Äî every numeric typeset for column-alignment
- **Slashed zero on mono**: `zero` enabled to disambiguate `0` from `O` in identifiers and connection strings
- **Single-family discipline**: Inter handles both display and body ‚Äî chromatic cyan does the work of typographic differentiation
- **Tight tracking on display**: -0.022em to -0.035em compresses the headline for confidence
- **Body 15px, not 16px**: matches dense-tooling rhythm; only hero-supporting copy uses 17px
- **`ss01` continuity**: alternate g feature shared between display and body for type consistency
- **No serif anywhere**: Turso's brand is committedly sans + mono ‚Äî no editorial-serif moments
- **Mono labels in micro size**: region pills use 11‚Äì12px mono with positive tracking for legibility at small sizes

## 5. Layout Principles

### Spacing System

- **Base unit**: 4px
- **Scale**: 2, 4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80, 96, 128
- **Density observation**: Turso's grid is **moderate density** ‚Äî denser than Render's spacious whitespace, looser than PlanetScale's terminal density. Sections at 64‚Äì96px, feature blocks at 16‚Äì24px

### Grid & Container

- **Page width**: 1280px max
- **Prose width**: 720px (documentation, marketing body)
- **Hero treatment**: full-bleed deep-teal canvas with the edge-map visualization spanning the hero
- **Feature grid**: 3-col at desktop, 2-col at tablet, 1-col at mobile

### Whitespace Philosophy

- Whitespace is functional ‚Äî generous between sections, dense within feature blocks
- The dark canvas itself functions as whitespace ‚Äî chromatic depth, not flat colour
- The edge-map visualization is the page's most decorative moment; everywhere else is restrained

### Section Cadence

- Hero (deep-teal canvas + edge-map) ‚Üí feature grid ‚Üí tinted teal feature panel ‚Üí product detail ‚Üí pricing ‚Üí footer
- The tinted teal panels function as section-break devices, not full backgrounds
- The cyan accent stays foregrounded ‚Äî it never anchors a section background

## 8. Interaction & Motion

### Easing

- **Standard**: `cubic-bezier(0.4, 0, 0.2, 1)` ‚Äî default
- **Emphasized**: `cubic-bezier(0.2, 0, 0, 1)` ‚Äî page transitions
- **Decelerate**: `cubic-bezier(0, 0, 0.2, 1)` ‚Äî tooltips

### Durations

- **Fast** (120ms): hover, focus
- **Standard** (200ms): card lift, button press
- **Slow** (320ms): modal enter, edge-map pulse cycle

### Per-Component Recipes

- **Button hover**: bg colour transitions over 120ms; subtle 1.005√ó scale
- **Card hover**: border opacity lifts, shadow intensifies, over 200ms
- **Edge-map dot pulse**: opacity 0.5 ‚Üí 1.0 over 1.5s ease-in-out infinite; staggered across regions
- **Status dot live pulse**: opacity 0.6 ‚Üí 1.0 over 1.2s ease-in-out infinite (disabled with reduced-motion)
- **Link hover**: underline grows from 0 ‚Üí 1px over 120ms

### Page Transitions

- Default: opacity-only fade over 320ms emphasized

### Reduced Motion

When `prefers-reduced-motion: reduce` is set:

- Edge-map dot pulses become static
- Status dot live pulses disabled
- Card hover scale removed
- All transitions reduce to opacity

## 12. Dark Mode & Theming

Turso is **dark-only**. The deep-teal canvas (`#0f1419`) and CRT-bright cyan accent are the brand ‚Äî they don't translate to a light surface (the cyan loses its glow, the chromatic story collapses). The dashboard product runs the same dark-only palette.

If a light variant were ever needed for documentation legibility:
- `bg`: `#0f1419` ‚Üí `#fafafa`
- `text`: `#ffffff` ‚Üí `#0f1419`
- `surface`: `#182228` ‚Üí `#f0f4f5` (light teal)
- `surface-tinted`: `#183134` ‚Üí `#cfe7e2` (light emerald)
- `brand` stays `#00face` (works on light, but loses the CRT-glow effect)

The recommendation is **don't**. The deep-teal canvas is the brand.

## 13. Lineage & Influences

Turso's chromatic identity is built around a single piece of brand-as-pun: the primary cyan is `#00face`, a hex that literally spells the word **FACE**. That self-referential gag is the brand's wink at the developers it courts, and the colour itself ‚Äî a CRT-bright cyan closer to `#00ffff` than to teal ‚Äî gives the page a glowing-edge-node feel.

The deep-teal canvas `#0f1419` reads as the dark side of an edge server, while the supporting emerald scale (`#1ebca1`, `#1e6952`, `#23765f`) gives Turso a chromatic ladder rather than a single-accent palette. Where Neon commits to mint and PlanetScale to blood-red, Turso commits to **edge cyan** ‚Äî bright, cool, glowing, immediately readable as "live replica online." Type is Inter, weights 600‚Äì700 for display, with mono carrying replication metrics. Buttons are 8px, cards 12px ‚Äî Vercel-class geometry with the chromatic spine doing the differentiation.

- **SQLite logo blue** ‚Äî https://www.sqlite.org ‚Äî glowing-cyan signal hue, modernised to CRT brightness
- **Neon** ‚Äî https://neon.com ‚Äî adjacent edge-database design language
- **Vercel** ‚Äî https://vercel.com ‚Äî Inter at 700 with negative tracking, 8px button radius
- **CRT phosphor displays** ‚Äî https://en.wikipedia.org/wiki/Cathode-ray_tube ‚Äî literal glowing-cyan-on-black aesthetic

## 14. Do's and Don'ts

### Do

- **Do** reserve `#00face` for primary action and live indicators ‚Äî it loses the "live replica online" semantic when used decoratively
- **Do** layer the emerald scale (`#1ebca1` ‚Üí `#23765f` ‚Üí `#1e6952`) for tinted panels and feature blocks
- **Do** keep the canvas at `#0f1419` ‚Äî pure black collapses the teal undertone, lighter teals lose the near-black anchoring
- **Do** use mono with `tnum` and `zero` for replication metrics, region pills, code samples
- **Do** use the edge-map visualization as the hero's decorative moment
- **Do** pair status colour-coding with text labels for accessibility
- **Do** honor `prefers-reduced-motion` ‚Äî disable edge-map and status-dot pulses
- **Do** keep section rhythm at 64‚Äì96px (denser than Render, looser than PlanetScale)
- **Do** use 8px button radius, 12px card radius (Vercel-class geometry)
- **Do** anchor body copy on `#9ba3a8` for muted, never below `#6b7882` for tertiary

### Don't

- **Don't** swap the cyan for a more conventional `#06b6d4` Tailwind cyan; Turso's `#00face` is brighter, slightly bluer, and self-referential
- **Don't** use the cyan and the emerald in the same component ‚Äî they belong to different layers (action vs surface)
- **Don't** add purple or magenta accents; the palette is intentionally cool-only, teal-to-cyan-to-green
- **Don't** flip to light mode; Turso's identity depends on the deep-teal canvas
- **Don't** apply heavy chromatic shadows; depth stays neutral so cyan reads as foreground
- **Don't** use serif anywhere ‚Äî Turso is committedly Inter + mono
- **Don't** use mono for body copy ‚Äî mono is for metrics, identifiers, code, region labels only
- **Don't** mix the brand cyan with a glow gradient outside the focus ring ‚Äî the cyan is flat-on-flat
- **Don't** introduce orange or red as primary chromatic accents ‚Äî semantic colours only
- **Don't** soften the dark canvas above `#1c2b34`; lifting too much loses the edge-node aesthetic
